1.1. Quais so os artefatos ?

  • Artefato: Documento Nomenclatura

1.1.1. Artefato: Documento Viso

  • um relato resumido com os principais tpicos em que o negcio a ser automatizado deve fornecer.
  • Normalmente, faz parte de um contrato de desenvolvimento de software.
  • Poder abordar: aspectos da tecnologia (linguagens, banco de dados, etc) que sero usados.
  • um documento voltado para os contratantes do software.

  • O diagrama de Caso de Uso (nvel 0) ou Pacote de Sistema baseia-se neste documento.

  • Apresenta de uma forma resumida o que o contratante (REQUISITANTE) deseja:

  • o que o contratado (DESENVOLVEDOR) quer:

  • Nasce na primeira reunio com os clientes de software (pessoas externas, internas, departamentos, etc).

1.1.2. Artefato: Documento Nomenclatura

  • Serve para criao de um padro nico de pr e ps-fixao para as palavras envolvidas em estruturas de cdigo.
  • Serve principalmente, para novos colaboradores ao se juntarem a equipe, serem treinados em um mesmo padro, sem existir vrios cdigos e estilos.
  • Cada empresa deve adotar uma notao que permita o encontro da identidade do seu formato de nomenclatura.

1.1.3. Artefato: Documento Glossrio

  • Traduz para as pessoas que no conhecem o software, todos os termos utilizados pelo usurio contratante e outros stakeholders.
  • Cada empresa adota o seu vocabulrio.
  • A finalidade deste documento agir como tradutor entre os participantes da anlise, do desenho, da codificao, dos testes e dos herdeiros do software e o vocabulrio particular da empresa.
Resumo
Os 3 documentos devem estar disponveis na nuvem da empresa para consultas de todos os participantes, inclusive os desenvolvedores de software que o tomaro como modelo para desenvolvimento de outros softwares.
-O Documento Viso o nico que exige assinatura do Cliente Contratante, pois ele delimita o escopo do sistema proposto.
-O Documento Nomenclatura deixa claro a forma de escrever o cdigo deve ser nica.
-O Documento Glossrio evita que entrem sem saber os significados dos termos utilizados no software desenvolvido.
